Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • A declaration is not case-sensitive.
  • The title element defines the title of the page. It is very important for SEO.
  • A SEO stands for Search Engine Optimization.
  • The style element defines CSS styles.
  • The base element is a base URL for the webpage.
  • A link connects an external resource to the HTML document.
  • The meta element is a character set, description, keywords, author, and viewpoint.
  • The script element embeds client-side scripts in HTML document.
  • A charset is a character set.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS is how content is presented to the user.
  • Inline CSS is applied to a specific HTML tag. Styling an element directly in HTML file.
  • Seperation of concerns is seperating code into different files based on use.
  • Declerationns contains 2 important components, the CSS property we want to apply & the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repo is every project in GitHub. It's a folder that contains all files and revision history.
  • Branch is a workspace where you can make changes that don't effect the live site.
  • Markdown (.md) is the way file are formatted in GitHub.
  • Commit changes - saving a new version of a file ypu're working on.
  • Pull request (PR) - asking for changes on branch to merge into another.
  • Merge is when PR is approved. Changes are merged into the main branch. They are live on the site.
  • An issue is a way to report new tasks/ content fixes.
  • Have to add git before a command in the terminal.
